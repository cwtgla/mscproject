Dataset folder problem down into <..>prec.txt files which contain values to be compressed. Files <...>prec_expected.txt contain the decimal values the unsigned bytes in the 24 bit structure should be. See below for reasoning on values chosen and working.


Working for:
--------------------------------------------------------------------------------------------------------------------------------------
	13 Mag 10 Prec. 2^13 = 8192 possible values, 8191 highest magnitude value. 2^10 = 1024 possible values, 1023 highest precision value. So multiply precision decimals by 1000 to get the value as an integer to 3 decimal places (we can represent 0-999 since with 4dp max is 1023 and we'd lose precision using it or extra overhead by seeing if we can store some values with extra precision and others without).

	7 Mag 16 Prec. 2^7 = 128 possible values, 127 highest magnitude value. 2^16 = 65536 possible values, 65535 highest precision value. So multiply precision decimals by 100,000 to get the value as an integer to 4 decimal places (we can represent 0-99,999 since with 5dp max is 65535 and we'd lose precision using it or extra overhead by seeing if we can store some values with extra precision and others without).
